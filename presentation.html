Εισαγωγή

Τα deepfakes είναι συνθετικά βίντεο όπου το πρόσωπο ή/και ο ήχος έχουν τροποποιηθεί με AI. Η ανίχνευσή τους αποτελεί ανοιχτό ερευνητικό πρόβλημα με σημαντικές εφαρμογές στη δημόσια πληροφορία και ψηφιακή ασφάλεια.

AUC

Το AUC μετρά αν το μοντέλο δίνει υψηλότερο score στα fake από ό,τι στα real, ανεξάρτητα από το threshold που θα διαλέξουμε.

AUC = 0.5 → τυχαίο
AUC = 0.7 → αδύναμο
AUC = 0.9 → ισχυρό
AUC = 1.0 → τέλειο
Για τη συνέχεια, η σύγκριση των μοντέλων βασίζεται στο AUC ως κύριο και αξιόπιστο κριτήριο.

Method

Το early fusion ενώνει τα δύο feature vectors πριν τον classifier. Το MLP βλέπει ταυτόχρονα audio και visual πληροφορία και μαθαίνει κοινή απόφαση.

x = concat(audio_vec, visual_vec)
logit = MLP(x)
score = sigmoid(logit)

Dimension alignment

CombinationPCA
HC + landmarks731 → 536
wav2vec2/HuBERT + landmarkspassthrough
SSL + Xception2048 → 768
Στο early fusion γίνεται concatenation πριν από το MLP. Η PCA αφορά μόνο το visual branch και μόνο όταν χρειάζεται alignment. Fit γίνεται μόνο στα train visual features για να αποφύγουμε leakage. Το MLP είναι κοινό, άρα αν το visual είναι αδύναμο μπορεί να επηρεάσει την εκπαίδευση.

Late Fusion

Audio MLP audio features → sigmoid score
↓
Video MLP visual landmarks → sigmoid score
↓
LateFusionCombiner s = w·sa + (1-w)·sv

Method

Το late fusion εκπαιδεύει πρώτα δύο ανεξάρτητα unimodal μοντέλα και συνδυάζει τις τελικές πιθανότητες. Δεν κάνει concat features και δεν χρειάζεται PCA.

Run 1
0.5 / 0.5
ίσο βάρος audio και video πριν την αξιολόγηση
Run 2
0.7 / 0.3
μεγαλύτερο βάρος στο πιο αξιόπιστο audio branch
Πλεονέκτημα: αν το video branch είναι αδύναμο, μπορούμε να μειώσουμε τη συνεισφορά του χωρίς να ξανασχεδιάσουμε τα features.
Στο late fusion δεν ενώνονται τα vectors. Έχουμε audio MLP και video MLP, παίρνουμε sigmoid scores και μετά τα συνδυάζουμε. Η υλοποίηση υποστηρίζει average και learned combiner, αλλά στα πειράματα χρησιμοποιούμε average. Στο Run 2 το weight γίνεται 0.7 υπέρ του audio.

Run 2 — Τρεις Αλλαγές

1. Sampler

WeightedRandomSampler υπήρχε ήδη στο Run 1. Στο training στοχεύει περίπου σε 5 fake : 1 real αντί για 42:1. Το test set μένει αμετάβλητο.

2. Loss weighting

Μετά τον sampler, τα fake παραμένουν περισσότερα στο batch. Επειδή fake=positive label, χρησιμοποιούμε pos_weight=0.2 για να μειωθεί η κυριαρχία τους στο loss.

5×0.2×loss(fake) + 1×loss(real) ≈ ισόρροπο gradient

3. Late fusion weight

Το Run 1 έδειξε καθαρό AUC gap: audio-only 0.91-0.93, video-only landmarks 0.571. Άρα στο Run 2 δίνουμε μεγαλύτερο βάρος στο audio.

s = 0.7·saudio + 0.3·svideo
Το 0.7 δεν είναι ακριβής μαθηματικός λόγος από το AUC. Είναι data-informed επιλογή: αρκετά υψηλή ώστε να κυριαρχεί το audio, αλλά όχι να μηδενίζει το video.
Η αλλαγή weight επηρεάζει μόνο τα 3 late fusion experiments.
Late wav2vec2+LM — Run 2 (AUC=0.907)
Τρεις αλλαγές. Ο sampler υπήρχε ήδη και φέρνει το training distribution κοντά σε 5 fake προς 1 real. Το pos_weight=0.2 αντισταθμίζει αυτό το residual ratio στο loss. Για late fusion, το AUC gap είναι σαφές: audio περίπου 0.91-0.93, video-only landmarks περίπου 0.57. Άρα το 0.7 είναι data-informed heuristic, όχι ακριβής τύπος από το AUC.

Future Work — Xception Visual Features

Γιατί τα Landmarks Αποτυγχάνουν

Landmarks = γεωμετρία. Deepfakes αφήνουν artifacts στο texture: blurring ορίων, inconsistent skin, rendering artifacts.

Πρόταση: Xception CNN

Pretrained ImageNet, depthwise separable convolutions → 2048-dim texture embedding ανά frame. Η αρχιτεκτονική που χρησιμοποιείται στη βιβλιογραφία deepfake detection (FaceForensics++).

Xception embedding: 2048-dim
+ Linear(2048 → 512) ← dimension alignment
→ Early / Late Fusion με audio features
Υλοποιηθηκε αλλα δεν δοκιμαστηκε λογω χρονικών περιορισμών — frame-by-frame extraction για 21k βίντεο απαιτεί σημαντικό GPU χρόνο.
